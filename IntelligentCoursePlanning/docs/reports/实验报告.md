## 二、数据结构设计

### 2.1 TimeSlot（上课时间段）

```cpp
struct TimeSlot {
    std::string day;    // Mon, Tue, Wed, Thu, Fri, Sat, Sun
    int beg;            // 开始节数，1-13
    int last;           // 持续节数，1-13

    int end() const;                    // 结束节数 = beg + last - 1
    bool overlaps(const TimeSlot&) const; // 节次重叠检测
    bool isWithin(day, beg, last) const;  // 范围包含检测
};
```

**设计要点**：一门课可能有多个 TimeSlot（如理论课 + 实验课），使用 `TimeSlotList = std::vector<TimeSlot>` 存储所有时间段。

### 2.2 CourseOffering（教学班）与 CourseBasic（课程基础）

```cpp
struct CourseOffering {  // course_info.csv 的一行
    std::string course_basic_ID;  // 课程基础序号（如 COMS0031121009）
    std::string course_sp_ID;     // 教学班序号（如 01）
    std::string semester;         // "秋季学期" / "春季学期"
    int recommended_term;         // 建议/最早修读学期 1-8
    std::string category;         // "专业必修课" / "学科基础课" / "专业选修课"
    double credit;                // 学分
    int beg_week, last_week;      // 周次范围
    TimeSlotList time_slots;      // 上课时间段
    // ...
    bool can_schedule_in_term(int term) const;  // 季节匹配判断
};

struct CourseBasic {  // 同一门课的所有教学班汇总
    std::vector<std::string> prereq_IDs;  // 先修课程（分号分隔解析）
    std::vector<CourseOffering> offerings; // 该课程的所有教学班
};
```

**设计要点**：
- `course_basic_ID + course_sp_ID` 组成教学班唯一键
- 秋季课程只能排 1/3/5/7 学期，春季课程只能排 2/4/6/8 学期
- `all_offerings` 存指针（`std::vector<const CourseOffering*>`），指向 `course_map` 中唯一数据源，避免副本不一致

### 2.3 Constraint（约束条件）

```cpp
struct Constraint {
    std::set<std::string> required_course_basic_IDs;       // 必修课集合
    std::set<std::string> elective_candidate_course_basic_IDs; // 选修候选集合
    std::map<int, double> max_credit_per_semester;          // 每学期学分上限
    double min_total_credit;                                 // 总学分下限
    double required_credit, elective_min_credit;             // 必修/选修学分要求
    std::vector<TimeBlock> avoid_time_blocks;                // 避让时段
    std::vector<TimeBlock> preferred_time_blocks;            // 偏好时段
    std::set<int> exchange_terms, internship_terms;          // 封锁学期
};
```

### 2.4 PrerequisiteGraph（先修关系图）

**数据结构**：邻接表 + 逆邻接表

```cpp
std::map<std::string, std::vector<std::string>> adj_list_;          // pre_id → [后继]
std::map<std::string, std::vector<std::string>> reverse_adj_list_;  // course_id → [先修]
```

**关键操作**：
- `build(dataset)`: 遍历 course_map，为有先修关系的课程建立有向边（仅包含数据集中存在的先修课）
- `topological_sort()`: Kahn 算法，O(N+E) 复杂度
- `has_cycle()`: 环检测，若存在则记录参与环的课程
- `check_missing_prerequisites()`: 检测先修课是否在数据集中存在
- `are_prerequisites_satisfied(id, completed)`: O(k) 判断 k 门先修课是否全部完成

## 三、规划算法思路

### 3.1 算法总览

```
Phase 0: 检查缺失先修课
Phase 1: 构建先修关系图，计算必修课先修链深度（DFS + 记忆化）
Phase 2: 按学期迭代(1→8)，贪心分配必修课
Phase 3: 从选修池补足选修学分
Phase 4: 方案验证（冲突/先修/学分全面检查）
Phase 4.5: 分析个性化时间偏好满足度
Phase 5: 生成规划摘要
```

### 3.2 先修链深度计算（DFS + 记忆化）

```
算法：compute_prerequisite_depth()

对每门必修课 id：
    dfs(id):
        if depth[id] 已计算：return depth[id]
        max_depth = 0
        for each 先修课 pre_id：
            max_depth = max(max_depth, dfs(pre_id) + 1)
        depth[id] = max_depth
        return max_depth

用途：排序时优先安排先修链深的课程（瓶颈课(高级课)优先）
```

### 3.3 贪心按学期分配（Phase 2）

```
for term = 1 to 8:
    if term 被封锁: continue
    current_credit = 0
    while 候选非空：
        candidates = get_candidates_for_term(term)   // 先修满足 + 季节匹配 + 可后移
        sort(candidates, 深度↓ → 学分↓)               // 瓶颈课优先
        for each id in candidates:
            if current_credit + credit > max_credit: continue
            offering = select_best_offering(id, term, existing)  // 选评分最高的教学班
            if offering != nullptr:
                加入当前学期
                current_credit += credit
                标记为已安排
```

**选教学班策略**：收集所有可行的教学班（季节匹配 + 时间不冲突），按软约束评分选择最优（偏好时段 +1 分，避让时段 -1 分）。

### 3.4 选修课补足（Phase 3）

```cpp
void fill_electives(result):
    for term = 1 to 8:
        for each 选修课 in 候选池:
            if 已安排: continue
            if 先修未满足: continue
            if 季节不匹配: continue
            if 学分超限: continue
            选择最优教学班（软约束评分）
            加入该学期
            if 选修学分 + 总学分均达标: return
```

### 3.5 时间复杂度分析

| 阶段 | 复杂度 | 计算来源 | 说明 |
|------|--------|---------|------|
| 先修图构建 | O(N + E) | 遍历所有课程（N）和先修边（E）建立邻接表 | N=1000 节点，E=664 边 |
| 深度计算 | O(N + E) | 每个节点 DFS 一次，记忆化避免重复 | 记忆化 map 保证每个节点只访问一次 |
| 贪心分配（Phase 2） | O(T × C × O) | 每学期（T）对候选课（C）遍历教学班（O） | T=8 学期，C≤22，O≤7 → 最多约 1232 次操作 |
| 选修补足（Phase 3） | O(T × E′ × O) | 每学期对选修候选（E′）遍历，每个也查 O 个教学班 | E′=149 门选修，每次选课也需遍历学期和教学班 |
| 验证 | O(T × K²) | 每学期对 K 门课两两比对冲突 | K=每学期课程数，最多约 16 门 |
| **总复杂度** | **O(N + E + T×C×O)** | N+E 图算法 + T×C×O 调度迭代 | 实际运行 < 0.5 秒 |

## 四、约束检查流程

### 4.1 时间冲突检查

```
输入：同一学期的选课列表 [off1, off2, ..., offN]

冲突条件（三维全满足才算冲突）：
  1. 周次重叠：! (end1 < beg2 || end2 < beg1)
  2. 星期相同：day_a == day_b
  3. 节次重叠：! (end1 < beg2 || end2 < beg1)

检查流程：
  for i = 0 to N-1:
    for j = i+1 to N-1:
      if has_conflict(off_i, off_j):
        // 周次不重叠 → 直接跳过
        // 周次重叠 → 检查所有 TimeSlot 组合
        for each slot_a in off_i.time_slots:
          for each slot_b in off_j.time_slots:
            if slot_a.overlaps(slot_b):
              记录冲突
```

### 4.2 先修关系检查

```
输入：已安排顺序的课程列表

for t = 1 to 8:
  for each course in semester[t]:
    if course 是必修课:
      for each prereq_id in course.prereq_IDs:
        if prereq_id 不在已完成集合中:
          报告先修缺失
    completed_set.insert(course.id)
```

### 4.3 学分检查

```
检查项：
  1. 每学期学分 ≤ max_credit_per_semester[t]
  2. 总学分 ≥ min_total_credit
  3. 必修学分 ≥ required_credit（所有必修课学分之和）
  4. 选修学分 ≥ derived_min_elective_credit_for_total

判断成功条件：
  result.success = unassigned.empty()
                && conflicts.empty()
                && total_credit >= min_total_credit
                && elective_credit >= derived_min
```

## 六、算法复杂度、适用范围、局限性和改进方向

### 6.1 算法复杂度

| 操作 | 复杂度 | 计算来源 | 实测时间 |
|------|--------|---------|---------|
| 数据加载（CSV 解析） | O(R) | 逐行读取 CSV 文件，每行做一次 split 和 trim | ~0.3s（3000 行） |
| 先修图构建 | O(N + E) | 遍历所有课程（N）和先修边（E）建立邻接表 | ~0.01s（1000 节点/664 边） |
| 深度计算 | O(N + E) | DFS 每个节点一次，记忆化避免重复遍历 | <0.01s |
| 贪心调度（必修） | O(T × C × O) | 每学期（T）对候选课（C）遍历所有教学班（O） | ~0.1s |
| 选修补足 | O(T × E × O) | 对候补选修（E）同样遍历学期和教学班，并且按学分排序 | ~0.05s |
| 验证 | O(T × K²) | 每学期（T）对 K 门课两两比对冲突 | <0.01s |
| **总计** | **O(N + E + T×C×O)** | N+E 图算法 + T×C×O 调度迭代 | **< 0.5s** |

其中：T=8 学期，C=候选课数(≤22)，E=候选选修数(≤149)，O=教学班数(≤6)，K=每学期课程数

### 6.2 适用范围

| 维度 | 当前能力 |
|------|---------|
| 课程规模 | 1000 门课程，3000 教学班 |
| 约束类型 | 学分上限、先修关系、时间冲突、时间偏好、封锁学期 |
| 专业类型 | 计算机、软件、数据科学、人工智能等 12 个专业 |
| 输入形式 | JSON 约束文件 + CSV 数据文件 |
| 输出形式 | 命令行报告 + JSON 导出 + Web 前端可视化 |

### 6.3 局限性

1. **贪心策略的局部最优**：当前算法采用贪心策略（每学期局部最优选择教学班），不保证全局最优的方案（如最小化晚课总数）。在最坏情况下可能错过更好的全局解。

2. **未考虑课程容量限制**：虽然数据中包含 `limits` 字段（选课人数上限），但当前算法假设每门课都能选上，未实现容量约束下的冲突解决。

3. **选修课匹配不够智能**：选修课的选取基于学分和先修条件，但没有根据学生兴趣方向（如 AI 方向、系统方向）进行课程推荐和匹配。

4. **单目标优化**：当前评分系统为线性加权（偏好+1，避让-1），未考虑多目标 Pareto 优化（如同时最小化冲突数、最大化偏好满足、均衡负载）。

5. **手动调整后不触发后端重规划**：前端支持手动选课/取消、学分约束调整和全面冲突检查（时间+先修+学分+偏好四项），但修改后的方案不会触发后端的 CSP 求解器重新优化剩余学期的课程分配。用户需重新运行 `course_planner.exe` 并加载新结果

### 6.4 改进方向

1. **引入回溯搜索**：在贪心无法安排所有必修课时，回溯到之前的学期重新选择教学班，提高可行解的搜索范围。

2. **多目标评分优化**：引入加权评分函数，考虑学分负载均衡度、时间偏好满足度、课程容量、学生兴趣等多维因素。

3. **增量重规划**：用户手动调整某学期方案后，自动重新规划受影响的后继学期（考虑已安排的课程作为固定约束）。

4. **课程推荐系统**：基于学生已选课程和专业方向，从选修池中智能推荐匹配度高的课程，结合课程之间的知识关联和职业方向。

5. **分布式/并行优化**：对于大规模课程池（10,000+），可使用并行拓扑排序和分布式贪心调度加速。

### A.1 Kahn 拓扑排序

```cpp
std::vector<std::string> kahn_sort(bool throw_on_cycle) const {
    std::map<std::string, int> in_degree;
    for (const auto& node : all_nodes_) in_degree[node] = 0;
    for (const auto& [pre, successors] : adj_list_)
        for (const auto& succ : successors) in_degree[succ]++;

    std::queue<std::string> q;
    for (const auto& [node, deg] : in_degree)
        if (deg == 0) q.push(node);

    std::vector<std::string> result;
    while (!q.empty()) {
        std::string node = q.front(); q.pop();
        result.push_back(node);
        for (const auto& succ : adj_list_[node])
            if (--in_degree[succ] == 0) q.push(succ);
    }

    if (result.size() != all_nodes_.size()) {
        // 有环：收集剩余节点
        for (const auto& [node, deg] : in_degree)
            if (deg > 0) cycle_nodes_.insert(node);
        if (throw_on_cycle) throw std::runtime_error("先修关系图中存在环路！");
    }
    return result;
}
```

### A.2 三维时间冲突检测

```cpp
bool has_conflict(const CourseOffering& a, const CourseOffering& b) {
    // 1. 周次不重叠 → 绝对不冲突
    if (!weeks_overlap(a.beg_week, a.last_week, b.beg_week, b.last_week))
        return false;
    // 2. 检查所有时间槽组合的 星期+节次 重叠
    for (const auto& slot_a : a.time_slots)
        for (const auto& slot_b : b.time_slots)
            if (slot_a.overlaps(slot_b)) return true;
    return false;
}
```

### A.3 教学班软约束评分

```cpp
// 收集所有可行的教学班
for (const auto& off : basic.offerings) {
    if (!off.can_schedule_in_term(term)) continue;
    if (!has_conflict_with_list(off, existing)) feasible.push_back(&off);
}

// 按偏好时段 +1 / 避让时段 -1 评分
for (const auto* off : feasible) {
    int score = 0;
    for (const auto& ts : off->time_slots) {
        for (const auto& pref : constraint.preferred_time_blocks)
            if (pref.contains(ts)) score++;
        for (const auto& avoid : constraint.avoid_time_blocks)
            if (avoid.contains(ts)) score--;
    }
    if (score > best_score) { best_score = score; best = off; }
}
```
