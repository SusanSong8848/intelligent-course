# 智能课程规划系统使用说明

本目录包含课程项目任务书、课程数据和样例约束文件。请先阅读 `项目说明/智能课程规划系统.pdf` 或 `项目说明/智能课程规划系统.md`，再根据任务要求完成系统设计与实现。

## 目录

- `项目说明/`：项目任务书，包含 PDF 和 Markdown 两种格式。
- `data/`：课程数据和样例约束。

## 必用数据

完成项目至少需要读取以下文件：

- `data/course_info.csv`：课程信息表。
- `data/course_time.csv`：课程上课时间表。

建议同时读取：

- `data/sample_constraints.json`：计算机科学与技术专业样例规划约束。

`sample_constraints.json` 不是唯一答案，而是一份可运行的样例输入。它说明了目标专业、必修课集合、选修池、学分上限、总学分要求、时间偏好等约束。学生可以基于它测试系统，也可以自行设计新的约束文件。

## 重要约定

- `sample_constraints.json` 中的 `target_course_scope.required_course_basic_IDs` 给出了样例任务的必修课集合，`required_categories_for_display` 用于展示这些课程所属类别。
- `recommended_term` 表示建议或最早修读学期，不是固定学期。课程可以后移到同季节的更晚学期，例如第 2 学期春季课可后移到第 4、6、8 学期。
- `teacher` 和 `classroom` 是合成字段，可用于展示或扩展检查，但项目基础要求主要关注先修、时间冲突、学分和类别约束。

## 可选资料

- `data/course_visualization.html`：课程数据可视化页面，可直接用浏览器打开。
- `data/major_profiles.json` 和 `data/major_profiles/`：多个专业的样例规划约束，可用于扩展测试。
- `data/major_profiles.html`：多专业约束可视化汇总。

## 提交提醒

项目最终提交应包含可运行程序、运行说明、使用的数据文件、图形化界面或可交互演示入口，以及实验报告。若使用 AI 辅助，应能解释核心算法、数据结构和关键代码。
